# XAEM Isoform Quantification & Bias Correction Pipeline

This document records the complete workflow for isoform quantification and bias correction using the XAEM tool, including environment preparation, input file specifications, step-by-step running commands, and key notes to ensure smooth execution. This workflow is tailored for Smart-seq2 (well-based platform) data.

## 1. Environment Preparation

First, load the runtime environment via Singularity to ensure a clean environment with complete dependencies, then activate the XAEM conda environment:

```bash
singularity exec --cleanenv --no-home -B /data/bioinf:/data/bioinf isoform_tools_v2.sif bash

# Activate XAEM conda environment
source /tools/miniforge3/bin/activate
conda activate XAEM
```

## 2. Official Documentation Reference

- XAEM official GitHub repository: https://github.com/WenjiangDeng/XAEM

- XAEM detailed user guide: https://www.meb.ki.se/sites/biostatwiki/xaem/

## 3. Input File Preparation

The following files need to be prepared in advance. Note that the X matrix is essential for bias correction and isoform quantification, and all input files must be consistent with the reference genome/transcriptome:

- **Fastq files**: Smart-seq2 sequencing data (paired-end, as specified in the command).

- **X matrix**: An essential object for bias correction and isoform quantification, provided as an RData file (e.g., X_matrix.RData).

- **Reference cDNA FASTA file**: Transcript sequence file (e.g., Homo_sapiens.GRCh38.cdna.all.fa) for indexing and alignment.

## 4. Running Commands

The XAEM workflow consists of four key steps: index transcript sequences, generate equivalence class table, create Y count matrix, and update X matrix & transcript expression using the AEM algorithm. Run the commands in sequence as follows:

### 4.1 Step 1: Index Transcript Sequences (TxIndexer)

Use TxIndexer (included in XAEM) to build an index for the reference cDNA FASTA file, which is required for subsequent sequence alignment and quantification:

```bash
/tools/XAEM-binary-0.1.2/bin/TxIndexer \
-t /data/bioinf/XAEM/data/Homo_sapiens.GRCh38.cdna.all.fa \
-o /data/bioinf/XAEM/data/TxIndexer_idx
```

### 4.2 Step 2: Generate Equivalence Class Table

Run XAEM to align paired-end fastq reads to the indexed transcript sequences and generate an equivalence class table. The command uses stream decompression for gzipped fastq files to avoid intermediate files:

```bash
/tools/XAEM-binary-0.1.2/bin/XAEM \
-i /data/bioinf/XAEM/data/TxIndexer_idx \
-l IU \
-1 <(gunzip -c /data/bioinf/scRNaseq_NSCLC/fastq/SRR10777215_1.fastq.gz) \
-2 <(gunzip -c /data/bioinf/scRNaseq_NSCLC/fastq/SRR10777215_2.fastq.gz) \
-p 4 -o ./SRR10777215
```

### 4.3 Step 3: Create Y Count Matrix

Use the provided R script to create the Y count matrix, which serves as input for the AEM algorithm. Specify the working directory, number of cores, and path to the X matrix:

```bash
Rscript /tools/XAEM-binary-0.1.2/R/Create_count_matrix.R workdir=$PWD  core=8  design.matrix=/data/bioinf/XAEM/data/X_matrix.RData
```

### 4.4 Step 4: Update X Matrix & Transcript Expression (AEM Algorithm)

Run the AEM algorithm (via R script) to update the X matrix and calculate final isoform/paralog expression levels, saving results as RData files for downstream analysis:

```bash
Rscript /tools/XAEM-binary-0.1.2/R/AEM_update_X_beta.R workdir=$PWD  core=8  design.matrix=/data/bioinf/XAEM/data/X_matrix.RData isoform.out=XAEM_isoform_expression.RData paralog.out=XAEM_paralog_expression.RData
```

### Key Parameter Explanations

- **TxIndexer**:
        

    - -t: Path to the reference cDNA FASTA file (transcript sequences).

    - -o: Path and prefix for the output transcript index file.

- **XAEM**:
        

    - -i: Path to the pre-built transcript index (generated by TxIndexer).

    - -l: Read orientation (IU = Illumina paired-end, forward-reverse orientation; adjust if using different sequencing platforms).

    - -1: Path to the first read of paired-end fastq files (uses stream decompression with `gunzip -c` for gzipped files).

    - -2: Path to the second read of paired-end fastq files (same stream decompression as -1).

    - -p: Number of threads used for parallel computing (set to 4 in this example).

    - -o: Path and prefix for the output equivalence class table.

- **Create_count_matrix.R**:
        

    - workdir: Working directory (uses `$PWD` to denote the current directory).

    - core: Number of cores used for parallel computing (set to 8 in this example).

    - design.matrix: Path to the X matrix RData file (essential for bias correction).

- **AEM_update_X_beta.R**:
        

    - workdir: Working directory (same as the one used in Create_count_matrix.R).

    - core: Number of cores used for parallel computing (set to 8 in this example).

    - design.matrix: Path to the X matrix RData file (must be the same as in the previous step).

    - isoform.out: Path and filename for the output isoform expression RData file.

    - paralog.out: Path and filename for the output paralog expression RData file.

## Notes

1. Path Description: All paths in this document are example paths; replace them with actual file paths (e.g., reference files, fastq files, output directories, XAEM binary/R script paths) when using to avoid running errors.

2. Environment Confirmation: Ensure the "XAEM" conda environment is successfully activated before running the commands. Also, confirm the paths to XAEM binaries (TxIndexer, XAEM) and R scripts are correct.

3. X Matrix Importance: The X matrix is essential for XAEM’s bias correction and isoform quantification; ensure the path to the X matrix RData file is correct and the file is not corrupted.

4. Fastq File Handling: The command uses `<(gunzip -c)` for stream decompression of gzipped fastq files, which avoids generating large uncompressed intermediate files. If using uncompressed fastq files, replace the `-1` and `-2` parameters with direct file paths.

5. Read Orientation: The `-l IU` parameter is for Illumina paired-end reads (forward-reverse orientation). Adjust this parameter according to your sequencing platform’s read orientation if needed.

6. Resource Adjustment: Adjust the `-p` (XAEM) and `core` (R scripts) parameters based on server CPU resources to balance running speed and resource occupation.

7. Working Directory Consistency: Use the same working directory (`workdir`) in both R scripts to ensure the pipeline can access intermediate files correctly.

8. Output Files: 
        - Transcript index (from TxIndexer) is saved in the specified `-o` directory.
        - Equivalence class table (from XAEM) is saved with the specified prefix (e.g., SRR10777215).
        - Y count matrix is generated in the working directory.
        - Isoform and paralog expression results are saved as RData files (specified by `isoform.out` and`paralog.out`) for further downstream analysis.
      

9. Reference Consistency: Ensure the reference cDNA FASTA file, X matrix, and fastq files are all based on the same reference genome (e.g., GRCh38) to avoid alignment and quantification errors.
