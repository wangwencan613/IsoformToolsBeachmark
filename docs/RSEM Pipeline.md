# RSEM Transcript Expression Quantification & Differential Analysis Pipeline

This document records the complete workflow for transcript expression quantification and differential analysis using the RSEM (RNA-Seq by Expectation-Maximization) tool, including environment preparation, input file requirements, step-by-step running commands, and key notes to ensure smooth execution.

## 1. Environment Preparation

First, load the runtime environment via Singularity to ensure a clean environment with complete dependencies, then activate the RSEM conda environment:

```bash
singularity exec --cleanenv --no-home -B /data/bioinf:/data/bioinf isoform_tools_v2.sif bash

# Activate conda environment
source /tools/miniforge3/bin/activate
conda activate RSEM
```

## 3. Input File Preparation

The following files need to be prepared in advance to ensure the analysis runs smoothly. This workflow is tailored for Smart-seq2 (well-based platform) data:

- Fastq files: Smart-seq2 sequencing data (paired-end, as specified in the command)

- Reference GTF file: Gene and transcript annotation file for the reference genome

- Reference FASTA file: Reference genome sequence file

- ngvector file: Generated by `rsem-generate-ngvector`, required for differential expression analysis with EBSeq

## 4. Running Commands

The RSEM workflow consists of four key steps: build RSEM reference, calculate transcript expression values, generate expression matrix, and perform differential expression analysis. Run the commands in sequence as follows:

### 4.1 Step 1: Build RSEM Reference

Construct the RSEM reference index using the reference FASTA and GTF files, with Bowtie2 as the alignment tool:

```bash
rsem-prepare-reference --gtf /data/bioinf/ref/38/Homo_sapiens.GRCh38.110.gtf \
       --bowtie2  \
       /data/bioinf/ref/38/GRCh38.p13.genome.fa \
       human_ensembl
```

### 4.2 Step 2: Calculate Transcript Expression Values

Quantify transcript expression from paired-end fastq files using the pre-built RSEM reference. Key parameters are explained below the command:

```bash
rsem-calculate-expression -p 8 --paired-end \
                        --bowtie2  --bowtie2-path ~/env/Scasa/bin \
                        --estimate-rspd \
                        --append-names \
                        --single-cell-prior \
                        /data/bioinf/scRNaseq_NSCLC/fastq/SRR10777215_1.fastq.gz \
                        /data/bioinf/scRNaseq_NSCLC/fastq/SRR10777215_2.fastq.gz \
                        /data/bioinf/RSEM/ref/human_ensembl  result/SRR10777215
```

### 4.3 Step 3: Generate Expression Quantity Matrix File

Combine expression results from multiple samples into a single expression matrix for downstream differential analysis:

```bash
rsem-generate-data-matrix result/SRR10777215.isoforms.results result/SRR10777216.isoforms.results result/SRR10777217.isoforms.results > result/isoform_matrix.txt
```

### 4.4 Step 4: Differential Expression Analysis

Perform differential expression analysis using EBSeq. First, prepare sample metadata, then generate bash scripts to run the analysis and filter results by FDR:

```r
# Read sample metadata
infor=read.csv('/hdd/data/dataset/Isoform/Datasets/metadata/GSE51254_meta.txt',header=T,sep='\t')
dim(infor)

# Define case and control groups, format sample IDs to match RSEM output files
case=infor[which(infor$class == 'case'),]
case$id=paste0(case$sample,'.isoforms.results')
control=infor[which(infor$class == 'control'),]
control$id=paste0(control$sample,'.isoforms.results')

# Generate script to create expression matrix for case vs control
path=paste0('cd /hdd/isoform/GSE51254/rsem/pair_end; rsem-generate-data-matrix ',paste(c(case$id,control$id),collapse = ' '),'>/hdd/isoform/GSE51254/rsem/pair_end/rsem_isoform_matrix_case_vs_control.txt')
write.table(path,'/hdd/isoform/GSE51254/rsem/rsem_DE_pair_end_case_vs_control.sh',row.names = F,col.names=F,quote=F,sep=',')

# Generate script to run EBSeq differential analysis and FDR control
path=paste("mkdir -p /hdd/isoform/GSE51254/rsem/pair_end/case_control","rsem-run-ebseq --ngvector /hdd/isoform/GSE52529/rsem-generate_human_gencode.ngvec  /hdd/isoform/GSE51254/rsem/pair_end/rsem_isoform_matrix_case_vs_control.txt 48,48 /hdd/isoform/GSE51254/rsem/pair_end/case_control/IsoMat.results",
           'rsem-control-fdr /hdd/isoform/GSE51254/rsem/pair_end/case_control/IsoMat.results  0.05  /hdd/isoform/GSE51254/rsem/pair_end/case_control/isoform.de.txt',sep='\n')
write.table(path,'/hdd/isoform/GSE51254/rsem/rsem_DE_pair_end_case_vs_control_2.sh',row.names = F,col.names=F,quote=F,sep=',')
```

### Key Parameter Explanations

- **rsem-prepare-reference**:
        

    - --gtf: Path to the reference GTF annotation file.

    - --bowtie2: Use Bowtie2 as the alignment tool for read mapping.

    - Last two parameters: Path to the reference FASTA file and prefix for the output RSEM reference index.

- **rsem-calculate-expression**:
        

    - -p: Number of threads used for parallel computing (set to 8 in this example).

    - --paired-end: Indicate that the input fastq files are paired-end.

    - --bowtie2: Use Bowtie2 for read alignment.

    - --bowtie2-path: Path to the Bowtie2 executable file.

    - --estimate-rspd: Estimate the read start position distribution (improves expression quantification accuracy).

    - --append-names: Append transcript/gene names to the output results (facilitates downstream analysis).

    - --single-cell-prior: Use a prior distribution suitable for single-cell RNA-Seq data (critical for Smart-seq2 data).

    - Last three parameters: Paths to paired-end fastq files, pre-built RSEM reference index, and prefix for output results.

- **rsem-generate-data-matrix**: Combine multiple RSEM isoform expression result files into a single tab-separated matrix.

- **rsem-run-ebseq**: Run EBSeq for differential expression analysis (requires ngvector file).

- **rsem-control-fdr**: Filter differential expression results by FDR (set to 0.05 in this example) to get significant DE transcripts.

## Notes

1. Path Description: All paths in this document are example paths; replace them with actual file paths (e.g., reference files, fastq files, output directories) when using to avoid running errors.

2. Environment Confirmation: Ensure the "RSEM" conda environment is successfully activated before running the commands. Also, confirm the path to Bowtie2 (--bowtie2-path) is correct.

3. Reference Construction: The RSEM reference index must be built first before running expression quantification. Ensure the reference FASTA and GTF files are consistent (e.g., both for GRCh38 genome).

4. Single-Cell Prior: The `--single-cell-prior` parameter is essential for Smart-seq2 (single-cell) data; do not omit it for single-cell analysis.

5. Resource Adjustment: Adjust the `-p` (threads) parameter based on server CPU resources to balance running speed and resource occupation.

6. Sample Replicates: For differential analysis, ensure there are multiple replicates in both case and control groups to improve result reliability.

7. ngvector File: The `--ngvector` parameter in `rsem-run-ebseq` requires a pre-generated ngvector file (via `rsem-generate-ngvector`); ensure this file is available and the path is correct.

8. FDR Threshold: Adjust the FDR threshold (0.05 in this example) in `rsem-control-fdr` based on your research needs.

9. Output Files: 
        - Expression quantification results (e.g., `.isoforms.results`) are saved in the specified output directory.
        - The expression matrix and differential analysis results are generated by the corresponding scripts for further downstream functional analysis.


10. Script Execution: After generating the bash scripts for differential analysis, run them via `bash script_name.sh` to execute the analysis in batch.
